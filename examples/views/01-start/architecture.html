아키텍처

IMCAT UI의 코어 구조와 모듈 시스템을 이해합니다.

코어 구조
프레임워크는 IMCATCore 싱글톤 클래스를 중심으로 20개 코어 모듈이 통합되어 있습니다.
// 전역 진입점
window.IMCAT = new IMCATCore();

// 코어 모듈 (빌드 시 포함)
├── dom.js          // DOM 조작 ($-like API)
├── router.js       // SPA 라우팅
├── event.js        // EventBus + EventEmitterMixin
├── state.js        // 반응형 상태 관리
├── storage.js      // localStorage 래퍼
├── api.js          // Fetch API 래퍼
├── security.js     // XSS 방어, 경로 검증
├── utils.js        // 유틸리티 함수
├── helpers.js      // 헬퍼 (폼, 다운로드, 스크롤)
├── template.js     // 템플릿 엔진
├── formatters.js   // 포맷터 (숫자, 날짜, 바이트)
├── config.js       // 설정 관리
├── loader.js       // 동적 모듈 로더
├── loading.js      // 로딩 오버레이
├── animation.js    // 애니메이션 유틸
├── form.js         // 폼 검증
├── shortcuts.js    // 단축 API (alert, confirm...)
├── auto-init.js    // data-imcat 자동 초기화
└── url.js          // URL 유틸리티
모듈 시스템
22개 확장 모듈은 IMCAT.use()로 필요할 때 동적으로 로드됩니다.
// 전체 모듈 로드 → 모든 export 반환
const Nav = await IMCAT.use('navigation');
// Nav = { Tabs, Accordion, Collapse, MegaMenu, TreeView }

// 하위 모듈만 로드 → 특정 클래스만 반환
const Tabs = await IMCAT.use('navigation/tabs');

// 모듈 로드 시 CSS도 자동 로드 (autoLoadModuleCSS: true)
// dist/modules/navigation.css → <head>에 자동 삽입
네임스페이스
코어 유틸리티는 네임스페이스 getter로 접근합니다.
// 네임스페이스 접근
IMCAT.utils.randomId('btn');     // 'btn-a1b2c3'
IMCAT.security.escape('<script>'); // '&lt;script&gt;'
IMCAT.helpers.copy('텍스트');      // 클립보드 복사

// 하위 호환: 직접 호출도 가능
IMCAT.isString('test');           // true
IMCAT.escape('<b>bold</b>');     // '&lt;b&gt;bold...'
메모리 관리
뷰 전환 시 destroy() 패턴으로 리소스를 자동 정리합니다.
// 인스턴스 등록 → 뷰 전환 시 자동 destroy()
const modal = new Modal({ ... });
IMCAT.view.registerInstance(modal);

// 단축 API는 자동 등록됨
const modal2 = await IMCAT.modal({ title: '제목' });
// → 뷰 이동 시 자동으로 modal2.destroy() 호출
